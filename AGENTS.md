# Repository instructions

## Commit and push convention

- Every commit created for this repository must use the exact subject format `faat(<scope>): <summary>` before it is pushed.
- Keep the literal lowercase prefix `faat`; do not silently change it to `feat`.
- Use a short, lowercase scope that identifies the affected area, such as `home`, `api`, `assets`, or `docs`.
- Write the commit scope and summary in English. Keep the summary concise and describe the completed change.
- Before committing and pushing, run the relevant lint and build checks and confirm that they pass.
